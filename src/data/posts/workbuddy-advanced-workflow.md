# WorkBuddy 进阶：多步骤任务编排与自动化工作流技巧

入门篇教你下单任务，实战篇带你看真实场景。

这篇讲进阶——**怎么让 WorkBuddy 干一整条流水线的活**。

现实工作很少是单一任务。更多时候是这样的：

> 调研市场 → 分析数据 → 写报告 → 做PPT → 发邮件通知

五个环节，串成一条链。手动做要一整天。用 WorkBuddy，可以自动化串起来。

## 一、核心思维：从"下单任务"到"编排工作流"

用 WorkBuddy 有三个层次：

| 层次 | 你做什么 | WorkBuddy 做什么 | 典型场景 |
|------|----------|------------------|----------|
| L1 单任务 | 下一个指令 | 完成一件事 | 写篇文章 |
| L2 多任务 | 下多个指令 | 每次完成一件事 | 写文章→改图→排版 |
| L3 工作流 | 描述整体目标 | 自主拆解、连续执行 | 从调研到交付全自动 |

大多数人停在 L1 和 L2。这篇重点讲怎么到 **L3**。

## 二、任务拆解：复杂任务的正确打开方式

WorkBuddy 再强，也不是万能的。遇到复杂任务，**你来拆解，它来执行**，效果最好。

### 错误示范

```
帮我做一份完整的竞品分析报告，要调研、要数据、要图表、
要PPT、要发邮件给老板。
```

太笼统。WorkBuddy 可能会做，但质量不可控，中间某个环节出问题你也不好定位。

### 正确示范：分步指令

**第 1 步：调研**
```
调研 [竞品A] 和 [竞品B] 的产品功能、定价、用户评价，
汇总成表格，数据标注来源。
```

**第 2 步：分析**（基于第 1 步的成果）
```
基于上面的调研表格，从功能完整度、价格竞争力、
用户口碑三个维度做对比分析，生成雷达图。
```

**第 3 步：写报告**（基于前两步）
```
把调研和分析整合成一份 2000 字的竞品分析报告，
结构：概述→功能对比→定价对比→用户评价→结论建议。
输出 Word。
```

**第 4 步：做 PPT**（基于报告）
```
基于这份报告，做一份 8 页的 PPT，
每页一个要点，配图表，风格简洁商务。
```

**第 5 步：发邮件**
```
写一封邮件给 boss@company.com，
主题"竞品分析报告"，正文 100 字概述结论，
附件是报告 Word 和 PPT。
```

### 为什么分步更好

1. **质量可控**：每一步做完你检查一下，不满意立刻改，不会等全做完才发现方向错了
2. **上下文连贯**：WorkBuddy 能基于上一步的成果继续做，不需要你重复说明
3. **可复用**：某个环节的指令可以存下来，下次换个竞品直接套用

## 三、链式执行：让 WorkBuddy 自己串流程

如果你信任 WorkBuddy 的能力，可以让它**自主编排**整条链。

秘诀是在指令里**明确流程和交付物**：

```
目标：完成 [某产品] 的市场调研并交付报告

请按以下流程执行，每完成一步向我确认后再继续：

步骤1：联网调研 [某产品] 所在行业的市场规模、主要玩家、
       增长趋势，输出调研笔记
步骤2：基于调研笔记，用 SWOT 分析该产品的竞争位置，
       输出分析文档
步骤3：整合调研和分析，写一份 3000 字的报告，输出 PDF
步骤4：基于报告做一份 10 页的汇报 PPT

每步完成后停下来，给我看成果，我说"继续"你再做下一步。
```

### 关键设计点

- **"每完成一步向我确认"** —— 这是保险阀，防止它跑偏
- **"输出XXX"** —— 每步都有明确的交付物，便于检查
- **流程顺序明确** —— 不让它自己猜先做什么后做什么

这种模式下，你是**项目经理**，WorkBuddy 是**执行团队**。你不用干活，但你把控节奏和质量。

## 四、自动化工作流：定时与批量

WorkBuddy 还能做更高阶的事——**定时执行**和**批量处理**。

### 场景：每日行业资讯摘要

假设你每天早上需要了解行业动态，手动刷新闻太费时。

**一次性的指令**可以这样写：

```
帮我做今日行业资讯摘要：

1. 搜索今天关于 [你的行业] 的最新新闻，取 10 条
2. 每条提取标题、来源、核心内容（50字）
3. 按重要性排序
4. 汇总成一份 Markdown 文档
```

**进阶：让它每天自动跑**

如果你的 WorkBuddy 支持定时任务功能，可以设置：

```
每天早上 8:00 执行以下任务：
- 搜索 [你的行业] 最新新闻
- 取前 10 条做摘要
- 发送到我的邮箱 / 保存到指定文件夹
```

一劳永逸。每天起床打开手机，行业简报已经在邮箱里了。

### 场景：批量处理文件

手头有 50 个 Excel 文件要统一处理？手动做要一天。

```
文件夹 /data 里有 50 个 Excel 文件，命名格式为
[地区]_[月份]_sales.xlsx。

请批量处理：
1. 每个文件读取后，按销售额降序排列
2. 取每个文件的前 5 名，汇总到一张总表
3. 总表按地区分组，算各地区月均销售额
4. 生成一张折线图展示各地区的月度趋势
5. 输出总表 Excel + 趋势图 PNG
```

WorkBuddy 会写一个 Python 脚本批量处理，50 个文件几分钟搞定。

## 五、4 个进阶技巧

### 技巧 1：先要大纲，再要全文

写长文档时，别直接让它写全文。先要大纲：

```
我要写一份 5000 字的 [主题] 报告，先给我大纲，
我确认后再写正文。
```

大纲不满意，改大纲的成本远低于改全文。

### 技巧 2：给示例，比讲规则管用

与其描述"我要什么风格"，不如直接给个例子：

```
帮我写产品介绍文案，风格参考这个例子：

[粘贴一段你喜欢的文案]

现在帮我为 [新产品] 写一段同样风格的介绍，300 字。
```

AI 从示例学习的效果，远好于从抽象描述学习。

### 技巧 3：让它自我检查

任务做完后，加一句：

```
完成后请自查：
- 数据有没有标注来源？
- 有没有错别字？
- 字数是否符合要求？
发现问题自行修正后再交付。
```

这一句能让交付质量提升一个档次。

### 技巧 4：建立你的指令库

好用的指令别用完就丢。存下来，形成你自己的**指令库**：

```
我的常用指令：
- 周报模板.md
- 数据分析模板.md
- 调研报告模板.md
- ...
```

下次换个主题，套模板就行。这是把个人经验**资产化**的过程。

## 六、什么时候用工作流，什么时候别用

工作流虽好，不是所有任务都适合。

**适合工作流的场景：**
- 环节之间有明确依赖（先调研才能分析，先分析才能写报告）
- 每个环节都有明确交付物
- 整体任务耗时较长（半小时以上）

**不适合的场景：**
- 单一简单任务（写个标题、翻译一句话）—— 直接做就行，别过度设计
- 创意类任务（写诗、想 slogan）—— 流程化反而限制创意，直接给需求让它发挥
- 不确定方向的任务 —— 先探索再谈流程

## 写在最后

从 L1 到 L3 的跨越，本质上是从"使用工具"到"管理系统"的思维升级。

你不是在用一个 AI 工具，你是在**搭建一条以 AI 为核心的生产流水线**。你是设计师，是项目经理，是质量把关人。

WorkBuddy 是执行者。你的编排能力，决定了这条流水线的上限。
